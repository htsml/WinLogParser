## 🗓 업데이트 기록

- Load 버튼 클릭 시 '컬럼 불러오기' 또는 '로그 불러오기' 중 선택 가능
- 컬럼을 먼저 불러오고 로그를 불러오면 실시간이 아닌 기존 로그 파일을 파싱하는 구조로 동작

### 📅 2025-05-16
**[Feature] 페이지 기반 로그 탐색 기능 추가**

#### 🔍 변경 목적
- 기존 자동 스크롤 방식은 실시간 로그 확인에는 좋았으나, 과거 로그 탐색이 불편
- 사용자가 특정 위치로 빠르게 이동하거나 원하는 페이지로 점프할 수 있는 기능이 필요했음

#### ✅ 구현 및 변경 사항
1. `pageSize`, `pageIndex` 구조 도입 (페이지당 줄 수 고정)
2. `pageIndex`를 기준으로 현재 보고 있는 페이지를 추적
3. `TotalPages` 계산 로직 추가 (전체 로그 수 기반)
4. 페이지 탐색 UI 추가
   - ◀ 이전 페이지 버튼 → `LoadPage(pageIndex - 1)`
   - ▶ 다음 페이지 버튼 → `LoadPage(pageIndex + 1)`
   - 페이지 번호 텍스트 입력 후 Enter → 해당 페이지로 이동
5. `LoadPage()` 함수 구현
   - 인덱스 유효성 검사, `RowCount` 및 `DataGridView` 갱신
   - UI 페이지 정보(`labelPageInfo`, `textBoxPage`) 표시 갱신
6. `AppendLog()`에서 실시간 로그 처리 로직 개선
   - 로그 추가 시 마지막 페이지에 있을 경우에만 자동 스크롤
   - `oldTotalPages`와 `newTotalPages` 비교를 통해 불필요한 스크롤 방지
7. `UpdatePage()` 함수에 `InvokeRequired` 조건 추가
   - 비동기 환경에서 발생 가능한 크로스 스레드 예외 방지
   - `BeginInvoke()` 사용으로 UI 안전 업데이트 수행
8. 장기 확장성 고려한 구조 설계
   - 로그 수가 수십만 줄 이상이더라도 `pageSize`만큼만 UI에 로딩
   - 필터, 검색, 오류 강조 등 추가 기능 확장을 위한 기반 마련
